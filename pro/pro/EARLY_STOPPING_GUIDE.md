# 早停机制使用指南

## 概述

早停（Early Stopping）是一种防止模型过拟合的训练优化技术。当验证集上的指标在连续多个epoch没有显著改善时，训练会自动提前停止，从而：
- 防止过拟合
- 节省训练时间和计算资源
- 获得更好的泛化性能

## 配置说明

在 `configs/best.yaml` 中的 `training.early_stopping` 部分配置早停参数：

```yaml
early_stopping:
  enable: true          # 是否启用早停
  patience: 50          # 容忍多少个epoch没有改善
  min_delta: 0.0001     # 最小改善阈值
  monitor: 'map50_95'   # 监控的指标名称
```

### 参数详解

#### `enable` (boolean)
- **默认值**: `false`
- **说明**: 是否启用早停机制
- **推荐**: 训练大型数据集或长时间训练时建议启用

#### `patience` (integer)
- **默认值**: `50`
- **说明**: 容忍多少个epoch没有改善
- **推荐**:
  - 小数据集: `30-50`
  - 中等数据集: `50`
  - 大数据集: `50-100`
- **注意**: 值越大训练越久，可能获得更好的性能，但也可能过拟合

#### `min_delta` (float)
- **默认值**: `0.0001`
- **说明**: 认为指标有改善的最小幅度
- **推荐**:
  - 标准设置: `0.0001`
  - 严格模式: `0.001`
  - 宽松模式: `0.0` (任意改善都算)
- **作用**: 避免因微小波动而重置等待计数器

#### `monitor` (string)
- **默认值**: `'map50_95'`
- **说明**: 用于早停的评估指标
- **可选值**:
  - `'map50_95'`: mAP@0.5:0.95 (COCO标准指标，推荐)
  - `'map50'`: mAP@0.5 (宽松指标，快速评估)
- **建议**: COCO评估使用 `map50_95`，快速评估可以使用 `map50`

## 工作原理

1. **监控指标**: 每次验证后检查指定的监控指标（默认为 `map50_95`）
2. **改善判断**: 如果当前指标 > 最佳指标 + min_delta，认为有显著改善
3. **重置计数器**: 有改善时，等待计数器重置为0，最佳指标更新
4. **增加计数器**: 无改善时，等待计数器加1
5. **触发早停**: 当等待计数器 >= patience 时，训练提前停止

## 恢复训练

早停机制支持断点续训，会自动保存和恢复早停状态：

1. **保存状态**: 每次保存检查点时，会保存早停的等待计数器和最佳指标
2. **恢复状态**: 从检查点恢复训练时，会恢复早停状态
3. **继续监控**: 恢复后继续从上次的状态进行早停监控

加载检查点时的日志：

```
早停状态已恢复: wait=28, best_metric=0.655000
已从 runs/train/checkpoint_epoch_130.pt 加载检查点
从 epoch 131 继续训练
```

## 最佳实践

### 1. 选择合适的patience值
- **训练初期**: 可以设置较大的patience（如100），给模型足够的时间收敛
- **训练后期**: 如果发现指标长时间不改善，可以减小patience
- **小数据集**: 建议patience=30-50，防止过拟合
- **大数据集**: 建议patience=50-100，给模型更多机会改善

### 2. 监控多个指标
虽然早停只监控一个指标，但建议同时关注：
- `map50_95`: 主要评估指标
- `map50`: 宽松评估指标
- 损失值: 训练损失和验证损失
- 精确率和召回率: 平衡评估

### 3. 结合TensorBoard
使用TensorBoard可视化指标变化，帮助判断是否需要调整早停参数：

```bash
tensorboard --logdir runs/train/best
```

### 4. 分阶段训练策略
1. **第一阶段**: 关闭早停或设置大patience，充分训练模型
2. **第二阶段**: 启用早停，防止过拟合
3. **微调阶段**: 使用小patience，快速找到最佳点

### 5. 验证集质量
- 确保验证集具有代表性
- 验证集应该足够大，避免指标波动过大
- 验证集应该与测试集分布一致

## 常见问题

### Q1: 早停触发太早怎么办？
**A**: 增大 `patience` 值或减小 `min_delta` 值

### Q2: 早停一直没有触发怎么办？
**A**: 
- 减小 `patience` 值
- 增大 `min_delta` 值
- 检查监控指标是否正确

### Q3: 早停后模型性能不佳怎么办？
**A**: 
- 恢复到早停前的最佳检查点
- 使用更小的学习率继续微调
- 调整数据增强策略

### Q4: 如何判断早停参数是否合适？
**A**: 观察训练日志中的：
- 等待计数器是否在patience范围内合理增长
- 最佳指标是否在合理范围内
- 训练曲线是否收敛平稳

### Q5: 可以同时监控多个指标吗？
**A**: 当前版本只支持监控单个指标，建议使用 `map50_95` 作为主要指标

## 禁用早停

如果不想使用早停，可以在配置文件中设置：

```yaml
early_stopping:
  enable: false
```

或者直接删除 `early_stopping` 配置块。

## 总结

早停机制是一个强大的训练优化工具，合理配置可以：
- ✅ 防止过拟合
- ✅ 节省训练时间
- ✅ 提高泛化性能
- ✅ 自动找到最佳模型

建议在大多数训练任务中启用早停，并根据数据集特点调整参数。
