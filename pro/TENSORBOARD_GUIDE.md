## 启动TensorBoard

### 基本启动命令

```bash
# 启动TensorBoard，指定日志目录
tensorboard --logdir=runs/train/runs

# 指定端口（默认6006）
tensorboard --logdir=runs/train/runs --port=6007

# 绑定到特定IP（远程访问）
tensorboard --logdir=runs/train/runs --bind_all
```

### 访问TensorBoard

启动后，在浏览器中访问：
- 本地访问：`http://localhost:6006`
- 远程访问：`http://服务器IP:6006`

## 训练阶段指标 (train/)

训练阶段的指标在每个epoch结束时记录，反映了模型在训练集上的表现。

### 1. train/total_loss - 总训练损失

**定义**：所有损失分量的加权和

**计算公式**：
```
total_loss = box_gain × box_loss + 
             cls_gain × cls_loss + 
             dfl_gain × dfl_loss + 
             obj_gain × obj_loss
```

**意义**：
- 衡量模型在训练数据上的整体预测误差
- 总损失应该呈下降趋势
- 用于监控训练是否正常进行

**正常范围**：
- 初始：通常在 10-50 之间（取决于模型规模和数据集）
- 收敛后：通常 < 5（优秀）或 < 10（良好）

**异常情况**：
- 损失不下降：学习率过小或数据有问题
- 损失变为NaN：梯度爆炸，需要降低学习率或使用梯度裁剪
- 损失突然上升：可能学习率过大或数据分布突变

### 2. train/box_loss - 边界框回归损失

**定义**：使用CIoU（Complete IoU）计算的边界框回归损失

**计算方式**：
```
box_loss = 1 - CIoU(pred_box, gt_box)
```

其中CIoU考虑了：
- 重叠面积
- 中心点距离
- 长宽比一致性

**意义**：
- 衡量预测框与真实框的匹配程度
- 反映模型定位目标的能力
- 值越小，定位越准确

**正常范围**：
- 初始：通常 0.1-0.5
- 收敛后：通常 < 0.05

**解读**：
- box_loss持续下降 → 定位能力提升
- box_loss波动较大 → 可能需要调整box_gain权重
- box_loss不下降 → 数据标注质量或模型架构问题

### 3. train/cls_loss - 分类损失

**定义**：使用二元交叉熵（BCE）计算的目标分类损失

**计算方式**：
```
cls_loss = BCE(pred_cls_score, target_cls)
```

**意义**：
- 衡量模型对目标类别的分类准确性
- 反映模型区分不同类别的能力
- 包括正样本（存在目标）和负样本（背景）

**正常范围**：
- 初始：通常 0.1-0.3
- 收敛后：通常 < 0.05

**解读**：
- cls_loss下降 → 分类能力提升
- cls_loss与box_loss同步下降是理想情况
- cls_loss过高 → 可能类别不平衡或特征提取不足

### 4. train/dfl_loss - 分布式焦点损失

**定义**：Distribution Focal Loss，用于优化边界框分布预测

**背景**：
- YOLOv8不直接预测边界框坐标，而是预测4个边在离散区间内的概率分布
- DFL Loss使预测的分布更接近真实值的分布

**计算方式**：
```
对于每个边，预测其在reg_max个区间内的分布
target = 边的真实距离
left_idx = floor(target)
right_idx = left_idx + 1
weight_left = right_idx - target
weight_right = target - left_idx
dfl_loss = weight_left × cross_entropy(left_idx) + 
           weight_right × cross_entropy(right_idx)
```

**意义**：
- 提高边界框定位的精度
- 使模型学习到更精细的边界框预测

**正常范围**：
- 初始：通常 0.5-2.0
- 收敛后：通常 < 0.5

**解读**：
- dfl_loss下降 → 边界框预测精度提升
- dfl_loss对训练后期精度提升很重要
- dfl_loss与box_loss相辅相成

## 验证阶段指标 (val/)

验证阶段的指标在每eval_period个epoch后计算，反映模型在验证集上的泛化能力。

### 1. val/map50 - mAP@0.5

**定义**：在IoU阈值为0.5时的平均精度（mean Average Precision）

**计算方式**：
```
对于每个类别：
  - 计算所有预测的置信度排序
  - 根据IoU=0.5判断预测是否正确（TP/FP）
  - 绘制Precision-Recall曲线
  - 计算曲线下面积作为该类别的AP
map50 = 所有类别AP的平均值
```

**意义**：
- 最常用的目标检测评估指标
- 衡量模型在相对宽松的定位要求下的检测能力
- IoU=0.5意味着预测框与真实框的重叠面积需达到50%以上

**正常范围**：
- 初始：通常 0.0-0.1
- 收敛后：
  - 简单数据集：0.8-0.95
  - 复杂数据集（如COCO）：0.5-0.7
  - 极难任务：0.3-0.5

**解读**：
- map50上升 → 检测能力提升
- map50稳定但train/loss仍在下降 → 可能过拟合
- map50突然下降 → 可能学习率过大或数据异常

### 2. val/map50_95 - mAP@0.5:0.95

**定义**：在IoU从0.5到0.95（步长0.05）的10个阈值下的平均精度均值

**计算方式**：
```
计算以下IoU阈值的mAP：
[0.50, 0.55, 0.60, 0.65, 0.70, 0.75, 0.80, 0.85, 0.90, 0.95]

map50_95 = mean(mAP@0.5, mAP@0.55, ..., mAP@0.95)
```

**意义**：
- 比map50更严格的评估指标
- 综合评估模型在不同定位精度要求下的表现
- COCO数据集的主要评估指标

**正常范围**：
- 初始：通常 0.0-0.05
- 收敛后：
  - 简单数据集：0.6-0.8
  - 复杂数据集（如COCO）：0.35-0.55
  - 极难任务：0.2-0.35

**解读**：
- map50_95通常比map50低0.15-0.3
- map50_95提升较慢，但更能反映真实定位精度
- map50高但map50_95低 → 定位精度不够，需要更多训练

### 3. val/AP_small - 小目标AP

**定义**：面积 < 32² (1024像素²) 的目标的平均精度

**计算方式**：
```
AP_small = mean(所有类别中，面积 < 32²的GT的AP)
```

**意义**：
- 衡量模型检测小目标的能力
- 小目标检测是目标检测的难点之一
- 对于航拍、远距离监控等场景很重要

**正常范围**：
- 通常比整体mAP低0.1-0.3
- 收敛后：
  - 良好：0.3-0.5
  - 一般：0.15-0.3
  - 较差：< 0.15

**解读**：
- AP_small显著低于整体 → 小目标检测困难
- 提升AP_small的方法：
  - 增加输入图像分辨率
  - 使用多尺度训练
  - 增强小目标的特征提取

### 4. val/AP_medium - 中目标AP

**定义**：32² ≤ 面积 < 96² (1024-9216像素²) 的目标的平均精度

**计算方式**：
```
AP_medium = mean(所有类别中，32² ≤ 面积 < 96²的GT的AP)
```

**意义**：
- 衡量模型检测中等大小目标的能力
- 在大多数数据集中占主要比例
- 通常比小目标好检测，比大目标难检测

**正常范围**：
- 通常接近或略低于整体mAP
- 收敛后：
  - 良好：0.6-0.8
  - 一般：0.4-0.6
  - 较差：< 0.4

**解读**：
- AP_medium应该是最高的（或接近最高的）之一
- 如果AP_medium过低 → 整体检测能力有问题

### 5. val/AP_large - 大目标AP

**定义**：面积 ≥ 96² (9216像素²) 的目标的平均精度

**计算方式**：
```
AP_large = mean(所有类别中，面积 ≥ 96²的GT的AP)
```

**意义**：
- 衡量模型检测大目标的能力
- 大目标通常最容易检测
- 在近距离、高分辨率场景下很重要

**正常范围**：
- 通常是最高的AP指标
- 收敛后：
  - 良好：0.7-0.95
  - 一般：0.5-0.7
  - 较差：< 0.5

**解读**：
- AP_large应该显著高于AP_small和AP_medium
- AP_large过低 → 模型可能严重欠训练或特征提取有问题
- AP_large与其他AP差距过大 → 可能需要平衡不同尺度的检测能力

### 6. val/f1 - F1分数

**定义**：精确率（Precision）和召回率（Recall）的调和平均

**计算公式**：
```
Precision = TP / (TP + FP)  # 预测为正的样本中，真正为正的比例
Recall = TP / (TP + FN)      # 真正为正的样本中，被正确预测的比例
F1 = 2 × (Precision × Recall) / (Precision + Recall)
```

**意义**：
- 综合评估模型的精确率和召回率
- 在精确率和召回率之间取得平衡
- 比单纯看精确率或召回率更全面

**正常范围**：
- 初始：通常 0.0-0.3
- 收敛后：
  - 优秀：0.8-0.9
  - 良好：0.7-0.8
  - 一般：0.5-0.7
  - 较差：< 0.5

**解读**：
- F1高 → 整体检测性能好
- Precision高但Recall低 → 模型保守，漏检多
- Recall高但Precision低 → 模型激进，误检多
- 根据应用场景调整置信度阈值来平衡Precision和Recall

## 参数解读指南

### 1. 理想的训练曲线

```
train/total_loss:  ↓↓↓ (持续下降，逐渐平缓)
train/box_loss:    ↓↓↓ (持续下降)
train/cls_loss:    ↓↓↓ (持续下降)
train/dfl_loss:    ↓↓↓ (持续下降)

val/map50:         ↑↑↑ (持续上升，逐渐平缓)
val/map50_95:      ↑↑↑ (持续上升，逐渐平缓)
val/AP_small:      ↑↑↑ (上升，但可能较慢)
val/AP_medium:     ↑↑↑ (上升)
val/AP_large:      ↑↑↑ (上升，通常最快)
val/f1:            ↑↑↑ (上升，逐渐平缓)
```

### 2. 训练与验证的关系

#### 正常情况：
- 训练损失持续下降
- 验证指标持续上升
- 两者都逐渐趋于平稳

#### 过拟合：
- 训练损失继续下降
- 验证指标停止上升或开始下降
- **解决方法**：增加数据增强、使用正则化、早停

#### 欠拟合：
- 训练损失很高且下降缓慢
- 验证指标很低
- **解决方法**：增加模型容量、训练更长时间、调整学习率

#### 训练不稳定：
- 损失曲线剧烈波动
- 验证指标上下跳动
- **解决方法**：降低学习率、增加batch size、使用梯度裁剪

### 3. 关键观察点

#### 早期阶段（0-30% epochs）
- 重点：损失快速下降，指标快速上升
- 这是模型学习基本特征的关键时期
- 如果这个阶段指标不上升，检查数据和学习率

#### 中期阶段（30-70% epochs）
- 重点：损失继续下降，速度变慢
- 指标稳步上升
- 开始关注不同尺度目标的AP差异

#### 后期阶段（70-100% epochs）
- 重点：指标趋于稳定
- map50_95的提升比map50更重要
- 关注小目标AP的改善

### 4. 指标优先级

根据任务需求，不同指标的优先级不同：

**通用检测任务**：
1. map50_95（最重要）
2. map50
3. f1

**小目标检测任务**（如航拍、远距离监控）：
1. AP_small
2. map50
3. map50_95

**高精度定位任务**（如医学影像、工业检测）：
1. map50_95
2. AP_large
3. map50

**实时检测任务**（需要平衡速度和精度）：
1. map50（在可接受的FPS下）
2. f1
3. map50_95

## 常见问题与解决方案

### 问题1：损失不下降

**可能原因**：
- 学习率过小
- 数据标注有问题
- 模型初始化问题
- 批量大小过大导致梯度不准确

**解决方案**：
```yaml
# 增加学习率
optimizer:
  lr: 0.01  # 从0.001增加到0.01

# 检查数据
# 确保标注格式正确，没有NaN值
```

### 问题2：损失变为NaN

**可能原因**：
- 学习率过大导致梯度爆炸
- 数值不稳定
- 损失函数实现有误

**解决方案**：
```python
# 在train.py中已经实现了梯度裁剪
nn.utils.clip_grad_norm_(self.model.parameters(), max_norm=10.0)

# 降低学习率
optimizer:
  lr: 0.001  # 从0.01降低到0.001

# 使用更小的权重衰减
optimizer:
  weight_decay: 0.0005  # 从0.05降低到0.0005
```

### 问题3：验证指标不上升

**可能原因**：
- 训练集和验证集分布差异大
- 模型过拟合训练集
- 验证集太难或太简单
- 预测阈值设置不合理

**解决方案**：
```yaml
# 增加数据增强
augmentation:
  mosaic: 1.0
  mixup: 0.15
  hsv_h: 0.015
  hsv_s: 0.7
  hsv_v: 0.4
  degrees: 10.0
  translate: 0.1
  scale: 0.5
  shear: 2.0
  perspective: 0.0
  flipud: 0.5
  fliplr: 0.5

# 调整置信度阈值
inference:
  conf_threshold: 0.001  # 使用更低的阈值进行评估
```

### 问题4：小目标AP很低

**可能原因**：
- 输入分辨率不够
- 小目标特征提取不足
- 训练数据中小目标样本少

**解决方案**：
```yaml
# 增加输入图像尺寸
training:
  image_size: [1280, 1280]  # 从640增加到1280

# 在augmentation中添加小目标增强
# 可以考虑使用Copy-Paste增强增加小目标样本
```

### 问题5：训练速度慢

**可能原因**：
- batch size过小
- 数据加载效率低
- 计算资源未充分利用

**解决方案**：
```yaml
# 增加batch size（如果显存允许）
training:
  batch_size: 32  # 从16增加到32

# 增加数据加载线程
training:
  num_workers: 8  # 从4增加到8

# 使用混合精度训练（需要修改代码）
# 添加：scaler = torch.cuda.amp.GradScaler()
```

### 问题6：map50高但map50_95低

**可能原因**：
- 边界框定位精度不够
- 模型对IoU敏感度低
- 训练时间不够

**解决方案**：
```yaml
# 增加训练epochs
training:
  epochs: 300  # 从150增加到300

# 调整损失权重
loss:
  box_gain: 7.5  # 增加box_loss权重
  cls_gain: 0.5
  dfl_gain: 1.5
```

## 使用TensorBoard的实用技巧

### 1. 对比多次实验

```bash
# 启动时指定多个日志目录
tensorboard --logdir=runs/exp1,runs/exp2,runs/exp3
```

在TensorBoard中可以通过下拉菜单对比不同实验的曲线。

### 2. 平滑曲线

在TensorBoard的左侧面板中，调整"Smoothing"参数：
- 0：显示原始数据
- 0.6-0.9：适度平滑，推荐
- 1.0：过度平滑

### 3. 查看特定指标

使用左侧面板的搜索框快速找到感兴趣的指标，例如：
- 输入`map`可以快速找到所有mAP相关指标
- 输入`loss`可以找到所有损失指标

### 4. 导出数据

点击TensorBoard右上角的下载按钮，可以将曲线数据导出为CSV文件，用于进一步分析或制作图表。

### 5. 设置关注点

对于重要的指标，可以点击星标将其标记为"关注"，这样它们会始终显示在顶部。

## 总结

TensorBoard提供了强大的可视化能力，帮助你：

1. **监控训练进度**：实时查看损失和指标的变化
2. **诊断问题**：通过曲线识别过拟合、欠拟合等问题
3. **优化模型**：对比不同超参数的实验结果
4. **报告结果**：生成专业的训练过程图表

**关键要点**：
- 关注趋势而非绝对值
- 训练损失和验证指标要一起看
- 根据任务需求调整指标优先级
- 定期保存和对比实验结果

**最佳实践**：
- 每个实验使用清晰的命名（如`runs/exp_lr001_bs32/`）
- 记录每次实验的超参数配置
- 定期检查TensorBoard，不要等到训练结束
- 使用平滑功能查看整体趋势，使用原始数据观察细节
